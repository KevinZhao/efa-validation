## 01:25 UTC — SPS 扫描（白名单机型，tc=4/3/2）

```bash
OUT=results/sps/$(date -u +%Y%m%dT%H%M%SZ)
mkdir -p "$OUT"
declare -A IR
IR[p5.48xlarge]="us-east-1 us-east-2 us-west-1 us-west-2"
IR[p5en.48xlarge]="us-east-1 us-east-2 us-west-1 us-west-2"
IR[p6-b200.48xlarge]="us-east-1 us-east-2 us-west-2"
IR[p6-b300.48xlarge]="us-west-2"

for TC in 4 3 2; do
  for itype in p5en.48xlarge p5.48xlarge p6-b200.48xlarge p6-b300.48xlarge; do
    aws ec2 get-spot-placement-scores --region us-east-1 \
      --instance-types "$itype" --target-capacity "$TC" \
      --single-availability-zone --region-names ${IR[$itype]} \
      --output json > "$OUT/${itype}-tc${TC}-az.json" 2>&1 || true
  done
done
```

**结果**：Ohio `use2-az2` p5en score=9 (tc=4) — 可用。Oregon `usw2-az3` 也 9 分。

## 01:30 UTC — Setup NVMe RAID0 on 4 nodes

Ohio LT v1 没 auto-LVM，需手动 mdadm 8-disk RAID0。**关键教训**：SSM multi-line heredoc 转义易坏，用 single-line inline 命令：

```bash
# setup NVMe via single-line SSM command (inline JSON params file)
cat > /tmp/sps-setup-inline.json <<'EOF'
{
  "commands": [
    "set -eux; ... (whole setup as one line) ..."
  ]
}
EOF
aws ssm send-command --region us-east-2 \
  --instance-ids i-xxx i-yyy i-zzz i-www \
  --document-name "AWS-RunShellScript" \
  --parameters file:///tmp/sps-setup-inline.json
```

## 02:25-02:45 UTC — R1c rate sweep（4/6/8/12/16）

脚本化成 `/tmp/rate-one.sh`（single-arg rate，避免 SSM output 24 KB 截断）：

```bash

R=$1
kubectl run r1c-bench-r$R-v2 --rm -i --restart=Never \
  --image=... --namespace=yanxi-validation \
  --overrides='{"spec":{"nodeSelector":{"topology.kubernetes.io/zone":"us-east-2b"},"tolerations":[...]}}' \
  -- python3 -m sglang.bench_serving --backend sglang \
  --base-url http://sglang-r1c-lb.yanxi-validation.svc:8000 \
  --dataset-name random --num-prompts 256 --request-rate $R \
  --random-input-len 1024 --random-output-len 512 \
  --tokenizer moonshotai/Kimi-K2-Instruct-0905
```

逐点跑（每点 60-90 s）：
- rate=6：**3532 tok/s** / TPOT 17 ms — 🏆 健康上限
- rate=8：2929 tok/s / TPOT 25 ms — decode 饱和起步
- rate=12：2646 tok/s / TPOT 47 ms — 全面饱和
- rate=16：3055 tok/s / TPOT 44 ms — queue plateau

写报告 `RATE_SWEEP.md`：结论 rate=6 是真正工作点，rate=12 看到的"prefill bottleneck"需要 ISL sweep 验证（后面发现是误判）。

Commit: `8baca30 R1c rate sweep — Kimi-K2 1P:3D ceiling ≈ 3500 tok/s`
